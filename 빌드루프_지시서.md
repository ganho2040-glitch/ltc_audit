## 1. 미션 & 완료 기준 (이 루프가 향하는 목표)

**미션:** `ltc_app.html`을 **실데이터 연쇄 누락 교정 도구**로 만든다.
수급자 + 로그된 기록 이벤트 → **끊긴 연쇄·임박 기한을 계산** → 보드 TOP3·통계·상세 타임라인·종에 표시.
더미 제거. 자료조사 규칙 기반. 현재 프레임("상시 이어서 할 일").

**시작점:** P5-2 구현(플랜 이미 승인됨) → P5-3 → P5-4.

**완료 기준 (DoD — 여기 닿으면 루프 정지하고 founder에게 보고):**
- [ ] 수급자별 기록 이벤트 입력·저장·타임라인 (P5-2)
- [ ] **핵심 연쇄 5종**이 로그 이벤트에서 *계산*되어 보드 TOP3·상세·종에 표시 (P5-3):
  - F03 상태변화→후속(상담/계획변경/결과평가)
  - F04 / I03 결과평가→계획 재작성 또는 유지근거(30일 기준)
  - F01 / Y01 욕구사정→급여제공계획 반영
  - F02 위험도평가→계획 반영
  - D 급여제공계획 설명·서명·공단 통보
  - (주기/기한 계산 포함. 출력 문구는 자료에서 가져옴.)
- [ ] "작성 완료로 표시"가 실제 연쇄를 해소 → 재계산 → 보드/종 갱신 (P5-4)
- [ ] 핵심 루프에서 더미(나할머니 등·정적 보드카드·거짓 gap) 제거, 실데이터로 대체
- [ ] 엔진 UI가 v3(쿨뉴트럴 토큰·라운드↓·구분선 그룹핑·tabular 숫자·상태칩)으로 보임 — 크림/카드남발 잔재 없음
- [ ] 각 단계 브라우저 동작 증거 + 커밋 존재

> ★ **P5-5(규칙 확장: 건강검진·직원교육·신규/이관 온보딩·평가당일 확인성·계획vs태그 등 자료 2·3차)는 이 자율 루프 밖.**
> DoD 도달 후 founder가 따로 결정한다. (n=1 위 과잉빌드 방지선.)

## 2. 그라운딩 & 프레임 (매 루프 적용)

- 엔진 내용(규칙·이벤트·연쇄·날짜·출력문구·우선순위) = **`엔진스펙_이어서할일.md`에서 가져온다(이미 우리 프레임으로 재구조화됨). 추측 금지.** 2·3차 규칙 상세가 필요할 때만 자료조사1·2 원본 참조.
- `rules.js` = 판정 로직 이식 자산.
- ★ **프레임 번역(필수):** 자료의 **엔진은 그대로 쓰되, 프레임은 폐기→번역**한다.
  자료에 깔린 "월말 리스크 감사 / 평가자 시선 모드 / 감점·환수 공포" 톤은 **쓰지 않는다.**
  현재 프레임 = **차분한 상시 "이어서 할 일."**

| 자료의 죽은 표현 | 산 표현(쓸 것) |
|---|---|
| 월말 점검 / 평가자가 본다 / 현지조사 대비 | 이어서 챙길 것 |
| 감점 위험 / 환수 / 부정청구 / 위반 | (공포 톤 제거 → 무엇을 이어서 하면 되는지만) |
| 리스크 감사 / 자체점검 점수 | 이어서 할 일 |
| ~확정 / 반드시 / 무조건 | 차분·진행형 (확인이 필요합니다 / 이어서 하세요) |

**디자인(v3) — 엔진 빌드에 녹인다(별도 스프린트 아님):**
- UI는 `디자인방향_v3_전문신뢰형.md` 기준. v3은 **락** — 피드백마다 갈아엎지 않는다(한 번 실행·멈춤).
- ★**첫 UI 터치 때 토큰 교체 1회 패스**(CSS 변수: 팔레트 크림→쿨뉴트럴 #F4F6F8/#FFFFFF + 딥틸 #0F4D45, 라운드 10~12, 그림자 틴트, 임상 상태색). v3 §10-1 = 이것만으로 인상 80% 전환. **바운디드 1슬라이스.**
- 이후 UI를 건드리는 슬라이스마다 v3 패턴: 카드 남발→구분선 그룹핑, tabular 숫자, 상태칩, 빈/로딩/에러. 본문 ≥15px·44px 유지.
- `landing.html`도 같은 쿨뉴트럴로 통일(후순위).

## 3. 루프 (슬라이스마다 반복)

1. **읽기:** SSOT + `진행기준` §4(다음 칸) + 이번 칸 관련 자료조사 규칙.
2. **계획(플랜 모드, 코드 X):** 접근법·데이터 형태·대상 규칙·출력문구(자료에서)·가정·리스크·**범위 한정**.
3. **자기점검(§5).** STOP 트리거(§6) 걸리면 → 멈추고 질문 적고 정지.
4. **구현:** 가장 작은 변경.
5. **검증:** 브라우저/실행 **실제 증거**("될 것" 아님). 안 되면 마지막 커밋 롤백.
6. **커밋**(작동 상태 박제) + `진행기준` §4 갱신(한 것 + 다음 칸, **덮어쓰기**).
7. 다음 슬라이스. **DoD까지 반복.**

> **P5-3 직후 1회 체크포인트:** 엔진 이식은 핵심 IP라, P5-3 커밋 후 **규칙 판정 로직 + 검증 증거를 founder에게 보고**하고 잠깐 멈춘다(이상 없으면 계속).
> 매 슬라이스 승인은 불필요 — 이 한 번만.

## 4. 검증 기준 (커밋 게이트)
"동작 증거"는 다음을 포함한다: 빈 상태·입력·새로고침 유지·계산 결과가 자료의 출력문구와 일치·완료표시 후 재계산·콘솔 에러 0.
증거 없이 커밋 금지. 증거가 안 나오면 = 아직 그 칸 안 끝남.

## 5. 자기점검 체크리스트 (커밋 전 매번)
- **북극성:** 연쇄 누락 교정인가? 복붙/"대신 써준다"로 드리프트 아닌가? → 의심되면 **HARD STOP**.
- **프레임:** 차분·"이어서 할 일". 공포·월말감사·확정 톤 0.
- **그라운딩:** 이 규칙 로직·날짜기준·출력문구가 자료/공식문서에 **실제로 있는가?** 추측이면 STOP.
- **스코프:** 이번 칸만. 자료 우선순위(1차 핵심 연쇄 먼저). 한꺼번에 다 X.
- **정직성:** 계산 전 값을 가짜로 표시 안 함(더미는 숨김, 거짓 정보 금지).
- **엔진 judge:** 판정 로직 임의 변경 금지(자료/공식문서 기준).
- **디자인:** 이 슬라이스가 UI를 건드리면 v3 적용(쿨뉴트럴 토큰·구분선 그룹핑·상태칩·tabular·빈/로딩/에러). v3은 락 — 재디자인 금지. 본문 ≥15px·44px 유지.

---

## 6. STOP하고 founder에게 묻는 트리거
- 자료가 규칙 로직/날짜기준에 **침묵하거나 모호** → 추측 금지, 질문.
- **스코프 분기**(이 규칙 1차에 넣나? 슬라이스가 너무 큰가?) → 질문.
- **프레임/카피가 자료에 없어 새로 지어야** → 톤 확인.
- **북극성 위반 의심**(복붙/대신써준다) → **HARD STOP**.
- 같은 문제 **2~3회 수정 실패** → 롤백·새 세션·보고.
- **엔진 judge 로직을 바꿔야 할 듯** → 질문(임의 변경 금지).
