## 0. 살린 것 / 폐기한 것

**살림(엔진):** 기록 이벤트 분류 · 연쇄 규칙 · 날짜/주기 로직 · 실무 출력 문구 · MVP 1·2·3차 우선순위.

**폐기·보류(우리 모델 아님):**
- "월말 리스크 감사 / 평가자 시선 모드" 프레임 → 폐기. 우리는 **상시 인앱 "이어서 할 일."**
- "감점·환수·부정청구 확정" 공포 톤 → 폐기. 차분·진행형으로 번역.
- 월말 PDF 감사 리포트 중심 구조 → **보류**(후순위 옵션, 엔진 아님).
- 무료 샘플 점검 CTA / 리포트 받기 → **세일즈·랜딩 영역**(엔진 아님).

**★입력 모델 경계 (북극성 보호 — strategic-lens 2026-06-20):**
우리는 **가벼운 신호("이 기록을 했다 + 날짜")에서 *연쇄*를 잡는다. 기록의 *내용*을 앱에 입력받지 않는다.**
- IN(가벼운 입력 = 우리 wedge): 노드 했음/날짜 → 연쇄 링크 누락(C1–C5). 날짜 기반(건강검진 일자·반기·통보일)도 가벼우면 확장 후보.
- OUT(무거운 내용 입력 = ERP·복붙 영역): 시간 불일치(R01, 분 단위 시간)·상담 필수값(C01, 상담 *내용* 입력·분석) 류. 케어포가 이미 가진 데이터를 센터장이 재입력하게 만들고 "대신 써준다/기록 보관" 드리프트로 빠짐 → 우리 기능 아님(케어포의 일).
- 경계 테스트: 규칙 추가 시 "센터장이 기록의 *내용*을 우리 앱에 타이핑해야 하나?" YES면 OUT(ERP), NO(했음/날짜만)면 후보(IN).

## 1. 북극성

- **문제:** 기록은 했는데, 이어져야 할 기록이 끊긴 걸 혼자서는 못 잡는다.
- **앱:** 끊긴 연쇄 단계를 *계산*해 "이어서 할 일"로 제시한다. **대신 써주지 않는다.**
- **프레임:** 차분·상시·진행형. (공포·월말·확정 톤 금지.)

## 3. 연쇄 규칙 — 핵심 5종 (= P5-3 최소 슬라이스)

각 규칙: **조건**(언제 "이어서 할 일"이 뜨나) · **출력 문구**(차분 프레임, 그대로 씀) · 원본코드(보조).

### C1 — 상태변화 → 후속조치  *(F03 / Y05)*
- **조건:** `change` 기록 있음(낙상·식사저하·수면저하·병원진료·수술·약복용 등) + 그 뒤 `counsel`/`plan 변경`/`result` 중 어느 것도 이어진 기록 없음.
- **출력:** "상태변화를 기록하셨어요. 이어서 — 방문상담·급여제공계획 반영·결과평가 중 하나로 이어가세요."

### C2 — 결과평가 → 계획 재작성/유지근거  *(F04 / I03 / Y11)*
- **조건:** `result` 있음 + 그 뒤 30일 내 `rewrite`(재작성 또는 "기존 계획 유지" 근거) 없음.
- **출력:** "결과평가를 마치셨어요. 이어서 — 30일 안에 계획을 재작성하거나 '상태변화 없음, 기존 계획 유지' 근거를 남기세요."

### C3 — 욕구사정 → 계획 반영  *(F01 / Y01)*
- **조건:** 평가연도 내 `needs` 있음 + 그 내용이 `plan`에 반영된 기록 없음. (신규 수급자는 급여개시일 기준.)
- **출력:** "욕구사정을 하셨어요. 이어서 — 확인된 내용을 급여제공계획에 반영하세요."

### C4 — 위험도평가 → 계획 반영  *(F02)*
- **조건:** 해당 반기 `risk` 있음(낙상·욕창·인지) + 그 결과가 `plan`/`service`에 반영된 기록 없음.
- **출력:** "위험도평가를 하셨어요. 이어서 — 낙상·욕창·인지 결과를 계획 또는 제공기록에 반영하세요."

### C5 — 계획 → 설명/서명/통보  *(D01·D02·D03)*
- **조건:** `plan` 있음 + `planNotify`(설명기록 / 확인서명·전자동의 / 공단 통보일) 중 빠진 것 있음.
- **출력:** "급여제공계획을 작성하셨어요. 이어서 — 수급자·보호자 설명, 확인서명, 공단 통보를 챙기세요."

> 정렬(보드 TOP3): 기한 임박(D-day) 우선 → 핵심 연쇄(C1~C5) → 그 외.

## 3-B. 감점 체크포인트 (리마인더 — *탐지 아님*, 행동 직전 안내)

연쇄(C1–C5)와 별개로, 특정 단계에서 *방식*을 틀리면 감점되는 디테일이 있다. 우리는 기록의 *내용/방식*을 입력받지 않으므로(§0 경계) **탐지하지 않고**, 그 단계를 작성·로그할 때 **리마인더**로 띄워 감점을 예방한다. (= 가벼운 입력 유지 — 리마인더이지 내용 입력 아님.)

- **R-서명 · 확인서명은 성함 전체를 정자로(타인이 알아볼 수 있게).**
  - 안내 문구(차분): "서명은 성함 전체를 *정자로* — 타인이 알아볼 수 있게요. 알아보기 힘든 싸인·도장·성만 적기는 감점될 수 있어요."
  - 적용 단계: **C5(계획 설명·확인서명)** 등 확인서명이 필요한 모든 단계.
  - 근거(검증됨, 원문): 평가매뉴얼 일반사항 10.용어설명 다.기타 3)서명 — *"서명 = 본인 고유의 필체로 자신의 이름을 다른 사람이 알아볼 수 있도록 쓰는 것."* + 재가급여 평가매뉴얼 다빈도 QA Q4(도장·알아볼 수 없는 서명 불인정, "정자로 된 전자서명"만 인정).
  - 프레임: 차분 안내(공포 0). 점수값 단정 금지("감점될 수 있어요" 정도).

※ 이런 체크포인트는 현장·QA에서 더 나온다 → **반드시 공식 문서 검증 후 추가(추측 0).** = ④ 커버리지·"컨설턴트 인 포켓" 어드밴티지의 일부.

## 3-C. 현장 보강 (네이버카페 실무 — 검증 입력 2026-06-20)

- **모니터링 = 급여제공결과평가 (동의어).** ERP마다 다르게 부름. 우리 `result` 노드 하나로 처리(중복 금지). 둘 중 하나만 하면 됨.
- **C2 해소 시점 재확인:** "결과평가 시기" 경보는 *결과평가를 해도 안 사라지고, 급여제공계획서 재작성(또는 "유지" 근거)까지 해야 해소.* → C2 판정과 정확히 일치. (이지케어 '카키색' = 6개월 도래 음영, 계획서 작성 시 사라짐.)
- **결과평가 주기 재확인:** 반기 1회(2x/년), 계획서 재작성 연 1회. 두 번 중 한 번은 결과평가 + "유지 근거"로 충분(무변화 증명 필요). → C2의 "재작성 또는 유지근거" 분기와 일치.
- **계획 적용기간 만료 임박 (후보 2차, 라이트 입력):** 계획서 적용기간 만료 = 재작성 시기(이지케어 '노란색'). 계획 작성 시 적용기간을 알므로 D-day 계산 가능. ★단 이지케어가 이미 색으로 표시 = 순수 기한 알림은 ERP와 겹침 → 우리 moat은 *연쇄·감점 디테일*이지 단순 기한 아님. 우선순위 낮춤.
- **신규 온보딩 순서(2차):** 계약 → 이용계획 열람신청 → 태그신청 → 기초욕구·위험도 → 욕구조사 → 계획서 작성·통보 → 일정배정. *신규는 첫 반기 결과평가 없음(하반기부터)* = I02 확인.
- **수급자 파일 편철 목차(2차·평가당일 확인성):** 계약(인정서·이용계획서·급여계약서·동의서) · 초기평가(욕구사정·초기상담·낙상/욕창 위험도) · 급여계획(계획서·설명동의·변경기록) · 급여제공기록(제공기록지·결과기록·특이사항).

## 3-D. 현장 보강 2 (네이버카페 — 2026-06-21)

> §3-C 후속. 아래는 **근거·후보·2차** 메모이며, 동결 엔진(C1–C5 쌍연쇄·판정·정렬·날짜)을 **바꾸지 않는다.** 표현 기록만.

- **급여 변경(시간·횟수) → 계획서 *종합의견란* 기록**(급여변경계약서 아님). 사유 = *기능상태 변화* vs *편의 요청* 구분. → **§3-B 리마인더 후보**(아직 활성 규칙 아님, 내용입력 아님 = §0 경계 유지).
- **연간 사이클 정규 순서(근거):** 상반기 결과평가 → 위험도 → 욕구조사 → 계획서 작성·통보 / 하반기 결과평가 → 위험도(무변화면 욕구조사·계획서 생략, **계획서 연 1회**). = 연쇄 *시퀀싱 근거*.
  - ▷ 정합: "계획서 연 1회"·반기 결과평가는 **§3-C·§4와 일치**. 단 이 순서는 *배경*이며 **엔진은 순서를 강제하지 않는 쌍연쇄**(needs·risk·result 각각 후속만 판정) — 규칙으로 코딩 금지.
- **이관·재이용(2차):** 타기관 이용 후 복귀(해지 안 함)면 **신규에 준함** — 결과평가 → 위험도 → 욕구조사 → 계획서.
  - ▷ 적용: 엔진 자동판별 아님 → 등록 시 `isNew`로 표기(§4 I02 예외와 연결). 우선순위 **2차**.
- **스마트장기요양앱(공식, 직교):** 일정은 시작 도래 시 표시, 계획 유효기간 연계(스케줄·태그·실시간 공식 거래 앱). **연쇄 빈틈은 공식앱·ERP 둘 다 안 봄 = 우리 영역.**
  - ▷ §1 북극성 + §3-C(moat = *연쇄·감점 디테일*, 단순 기한 아님)와 **일치·보강.**

표현 기록만. **엔진 0 변경.**
