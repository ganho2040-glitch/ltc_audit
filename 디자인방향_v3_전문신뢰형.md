# 디자인 방향 v3 — "전문 신뢰형" (Professional / Clinical-credible)

> 2026-06-20 확정. v2 "딥틸 따뜻한 행정 SaaS"를 **폐기**하고 이걸로 교체.
> 폐기 이유: 실제 센터장 피드백 — "너무 단순·전문성 떨어짐." 진단: 크림 베이스 + 카드 남발 + 넓은 여백 + 다정한 톤 = *소비자 앱*처럼 보였음. 우리 타깃은 임상 B2B.
> 기준: premium-frontend-design **Section B(올라운더, 전문 프리미엄)** + 보편 규칙. Section D(세리프/크림 에디토리얼)는 폐기.
> ★이 문서가 디자인 단일 기준(SSOT §10이 여기를 가리킴). 디자인은 이미 두 번 갈렸다(세리프/크림→딥틸→여기). **이게 락(lock)이다. 피드백마다 갈아엎지 않는다 — 한 번 실행하고 멈춘다.**

---

## 0. 한 줄 정체성
"센터장이 *전문가가 만든 의료 소프트웨어*로 신뢰하되, 50~60대가 모바일에서 또렷이 읽는다."
레퍼런스 앵커: **이미지2(회진도우미)의 명료함 + 현대 전문 SaaS 대시보드의 폴리시.**
NOT 이미지1(병원 벽면 현황판 = 1인 모바일엔 과밀). NOT 이미지3·4(소비자 다정함).

## 1. 디자인 노브 (skill 기본값을 타깃에 맞게 의도적으로 낮춤)
- VARIANCE 3 (예측가능·차분 = 신뢰 + 고령 가독성). 아티스틱 금지.
- MOTION 3 (조용·접근성. 화려한 전환 금지).
- DENSITY 6 (데이터 도구답게 화면을 채움. 단 ≥15px·44px 유지). 텅 빈 데모 느낌 탈출.

## 2. 팔레트 (가장 큰 전환: 크림 → 쿨 뉴트럴)
- 캔버스: `#F4F6F8` (쿨 라이트 그레이, 임상-청결감)
- 표면/카드: `#FFFFFF`
- 구분선: `#E6EBF0` / 테두리 `#DCE3EA` (1px가 구조의 기본)
- 브랜드 액센트(단 하나, 채도<80%): 딥틸 `#0F4D45` (dark `#083A34` / soft `#E6F0EE`) — 브랜드 연속성 유지, 쿨 뉴트럴 위에서 전문적으로 읽힘
- 텍스트: ink `#0F172A`(순흑 아님) / sub `#475569` / muted `#94A3B8` (쿨 그레이)
- 상태(임상적·정밀, 파스텔 아님):
  - 위험 `#B42318` / 배경 `#FEF3F2`
  - 주의 `#B54708` / 배경 `#FFFAEB`
  - 정상 딥틸 `#0F4D45` / 배경 `#ECF6F3`
  - 정보 `#475569` / 배경 `#F1F5F9`
- 규칙: 한 프로젝트 = 한 팔레트. 그라디언트·네온·글로우·순흑·과채도 금지.

## 3. 타이포 (Pretendard 유지 — 한글 전문 폰트, "AI 티" 아님)
- 위계 강하게. 숫자는 **tabular/lining**로 데이터 강조.
- 타이틀 22~26 Bold(트래킹 살짝 타이트) / 섹션 16~17 SemiBold / 본문 15~16(고령 ≥15) / 캡션 12.5~13.
- 데이터·상태 숫자는 굵게 강조 = "데이터 도구" 신호. 세리프 금지(대시보드).

## 4. 레이아웃 & 머티리얼 (전문성의 핵심 레버)
- ★**카드 남발 중단.** 밀집 영역은 *구분선 그룹핑*(border-t / divide-y) + 네거티브 스페이스로 구조화. 카드/엘리베이션은 *진짜 위계*에만(예: 먼저 챙길 TOP3, 잡아낸 gap). → Linear/전문 SaaS 느낌, 소비자 카드앱 탈출.
- 라운드: 카드 10~12 / 칩 8 (v2의 16에서 축소 = "소프트웨어"답게).
- 그림자: 아주 미묘, 배경 hue로 틴트 — `0 1px 2px rgba(15,23,42,.06)`. 구조는 그림자보다 1px 테두리로.
- 화면마다 상단 앱바(타이틀 + 종) = "시스템" 느낌.
- 밀도: 패딩 타이트(14~16) + 화면당 정보↑. 단 44px 터치·≥15px 본문 절대 유지.

## 5. 컴포넌트
- 리스트 행: 흰 배경 위 divide-y 그룹, 좌측 상태 색선(위험/주의/정상), 우측 상태 칩 + chevron.
- 상태 칩/도트: 정밀·일관 색코드(임상 가독성).
- 미니 데이터 비주얼: 카운트 배지, 촘촘한 임상 타임라인, "이번 주" 요약 스트립 = 전문성 cue.
- 버튼: 솔리드 딥틸, :active에 scale(.98)/-1px. 라디우스 10.
- 폼: 라벨 위, 헬퍼/에러 마크업, gap. 빈 상태·로딩(스켈레톤)·에러 상태 구현.
- 아이콘: 일관 라인 아이콘 시스템(균일 stroke, 임상적). 이모지 아이콘 금지.

## 6. 카피 톤
평이한 한글(센터장 언어) 유지하되 *조금 더 또렷·정밀하게*("다정한 소비자" 톤 ↓, "또렷한 전문 안내" ↑). 차갑지 않게. 공포 마케팅 금지.

## 7. 유지 / 폐기
- 유지: Pretendard, 딥틸 브랜드, 북극성 UX(이어서 할 일/연쇄), 모바일 우선 390~430, 신뢰 문구(localStorage), 44px·≥15px.
- 폐기: 크림 베이스, 카드 남발, 큰 라운드/아늑함, 파스텔 소프트, 소비자 다정함, 과한 여백("텅 빈" 인상).

## 8. 금지 (보편 + 확장)
이모지 아이콘 · 그라디언트 · 네온/글로우 · heavy shadow · 순흑 · 과채도 액센트 · 그라디언트 텍스트 · Inter/Roboto · 대시보드 세리프 · 3열 동일카드 · 가짜 placeholder 이름 · 둥근 가짜 지표 · 커스텀 커서 · **그리고 토이/소비자-아늑 미감.**

## 9. 가드레일 (타협 불가)
- 50~60대 가독성: 본문 ≥15px, 44px 터치, 고대비, 차분·예측가능(낮은 변주). 전문성은 *폴리시·밀도·크래프트*로 — 작은 글씨·과밀로가 아니라.
- **실데이터(P5 엔진)가 전문성의 절반을 자동 해결**한다(텅 빈 데모라서 단순해 보임). → v3은 별도 리디자인 스프린트가 아니라 *엔진 빌드 중 UI 손댈 때* 입힌다.
- ONE 방향. 한 번 실행. 멈춤. 디자인-바이-커미티 금지.

## 10. 실행 순서 (엔진 빌드에 녹임)
1. 토큰 교체(팔레트 크림→쿨뉴트럴, 라운드↓, 그림자 틴트) — CSS 변수만 바꿔도 80% 인상 전환.
2. 카드 남발 → 구분선 그룹핑으로 리팩터(밀집 리스트부터).
3. 타이포 위계·tabular 숫자.
4. 상태 칩·아이콘 시스템 일관화 + 미니 데이터 비주얼.
5. 빈/로딩/에러 상태.
→ landing.html도 같은 쿨 뉴트럴 팔레트로 통일(제품 일관성).
